# 项目定制化开发规则 (AGENTS.md)

1. **方案对齐后方可改动代码**：如果用户提出了一堆 Bug 反馈或修改意见，且未明确指示具体的实现技术方向，**必须先完成技术分析，编写或更新 `implementation_plan.md` 方案并征得用户明确同意后方可改动本地代码**。
2. **意见迭代优先更新 Plan**：在评审方案或开发过程中，如果用户针对同一组 Bug 提出了补充意见或微调要求，必须先将意见修改、合并并沉淀至 `implementation_plan.md` 方案中重新确认，而不能在方案未达成一致时直接在本地修改代码。
3. **严禁自动推送 Git**：绝不能在未获得用户指示前擅自执行 `git commit` 或 `git push` 等任何 Git 修改与推送指令。一切提交必须交由用户决定或在用户发出明确指令时协助执行。
